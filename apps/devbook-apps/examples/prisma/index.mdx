## <Run /> Pagination and Sorting
###  Sort

Prisma Client supports [filtering and sorting](https://prisma.io/docs/concepts/components/prisma-client/filtering-and-sorting) with the `where` and `orderBy` query options respectively.

Use [orderBy](https://prisma.io/docs/reference/api-reference/prisma-client-reference#orderby) to sort a list of records or a nested list of records by a particular field or set of fields.

<Highlight lines="5-8">
You can **sort by properties of a relation**. 
Property used for sorting is defined by <Input entry="property" values={["published: true", "published: false"]} /> and the <Input entry="order" values={["createdAt: \"asc\"", "createdAt: \"desc\""]} /> defines the sorting order.
</Highlight>
<Output type="array" position={1} noContentLabel="Waiting for data" >
<State entry="property" transform={{ "published: true": "Published", "published: false":"Non-published" }}/> posts in <State entry="order" transform={{ "createdAt: \"asc\"": "ascending", "createdAt: \"desc\"":"descending" }}/> order:
</Output>

<Highlight lines="11-13">
In 2.19.0 and later, you can sort by the **count of related records**. <Input entry="count-order" values={["_count: \"asc\"", "_count: \"desc\""]} /> defines the sorting order.
</Highlight>
<Output type="array" position={2} noContentLabel="Waiting for data">

Authors in <State entry="count-order" transform={{ "_count: \"desc\"": "descending", "_count: \"asc\"": "ascending" }}/> order by the number of posts:
</Output>
</div>

<Code file="index.js" onRun={() => "node /code/index.js"}>
{`import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

const posts = await prisma.post.findMany({
  where: { published: true },
  orderBy: { createdAt: "asc" },
});
console.log(JSON.stringify(posts, null, 2));

const prolificAuthors = await prisma.user.findMany({
  orderBy: { posts: { _count: "desc" } },
});
console.log(JSON.stringify(prolificAuthors, null, 2));
`}
